# 프로젝트 개발 환경 및 초기 셋업 가이드

이 프로젝트(`koreanroad`)에 처음 합류하시거나 새로운 PC에서 작업을 시작하셔야 할 때를 위한 **개발 환경 요약 및 초기 셋업 가이드**입니다.

## 1. 현재 프로젝트 개발 환경 요약

이 사이트는 최신 프론트엔드 기술 스택으로 구성된 정적 웹 사이트(Static Export)입니다.

*   **코어 프레임워크:** Next.js 16.2 (App Router 기반)
*   **UI 라이브러리:** React 19.2
*   **언어:** TypeScript 5.x
*   **스타일링:** Tailwind CSS 4.x (PostCSS 사용)
*   **빌드 출력:** `output: "export"` 설정으로 정적 파일(HTML/CSS/JS) 형태로 빌드됨
*   **런타임(권장):** Node.js v20 이상 (`package.json`의 `@types/node: ^20` 기준)

## 2. 개발 환경 초기 설정 가이드 (로컬 셋업)

새로운 PC에서 이 프로젝트를 열고 개발을 시작하기 위한 순서입니다.

### Step 1: 필수 프로그램 설치
가장 먼저 개발에 필요한 기본 도구들이 설치되어 있는지 확인합니다.
1.  **Node.js 설치**: `Node.js 20.x LTS` 버전 이상을 설치합니다. (공식 사이트: nodejs.org)
    *   *확인 방법:* 터미널에서 `node -v` 및 `npm -v`를 입력하여 버전이 잘 나오는지 확인합니다.
2.  **IDE(에디터) 설치**: **Visual Studio Code(VS Code)** 사용을 권장합니다.
    *   추천 확장 프로그램(Extensions): `ESLint`, `Tailwind CSS IntelliSense`, `Prettier` (필요시)
3.  **Git 설치**: 소스 코드 관리를 위해 Git을 설치합니다. (공식 사이트: git-scm.com)

### Step 2: 프로젝트 폴더 준비 및 패키지 설치
1.  **터미널(명령 프롬프트 또는 VS Code 터미널)** 을 엽니다.
2.  프로젝트 소스가 아직 없다면 저장소에서 클론(Clone) 받습니다.
    ```bash
    git clone [저장소 URL]
    ```
3.  프로젝트 폴더로 이동합니다.
    ```bash
    cd koreanroad
    ```
4.  프로젝트 구동에 필요한 라이브러리(Dependencies)를 설치합니다. 이 명령어는 `package.json`을 읽고 필요한 파일들을 `node_modules` 폴더에 다운로드합니다.
    ```bash
    npm install
    ```

### Step 3: 로컬 개발 서버 실행
설치가 완료되면 개발 모드로 서버를 띄워 실시간으로 작업 결과를 확인합니다.

1.  터미널에 아래 명령어를 입력합니다.
    ```bash
    npm run dev
    ```
2.  명령어가 정상적으로 실행되면 터미널에 `http://localhost:3000` 형태의 주소가 나타납니다.
3.  크롬(Chrome) 등 웹 브라우저를 열고 해당 주소로 접속하면 사이트 화면을 볼 수 있습니다. 코드를 수정하고 저장하면 브라우저가 자동으로 새로고침(Hot Reload) 됩니다.

### Step 4: 배포를 위한 빌드 (선택 사항)
작업이 끝난 후, 실제 서버나 호스팅에 올릴 파일을 만들고 싶을 때 사용합니다.

```bash
npm run build
```
*   현재 `next.config.ts`에 정적 내보내기(`output: "export"`)가 설정되어 있기 때문에, 이 명령어를 실행하면 루트 경로에 **`out`** 이라는 폴더가 생성됩니다. 
*   이 `out` 폴더 안의 내용물(HTML, CSS, 이미지 등)을 웹 서버(S3, GitHub Pages, Vercel 등)에 그대로 업로드하면 서비스가 배포됩니다.

---

**💡 요약 팁:**
평소 개발하실 때는 터미널을 열고 **`npm run dev`** 만 입력해둔 상태에서 코딩을 진행하시면 됩니다!
